# GHI Forecast Risk on Raspberry Pi

This project downloads MeteoSwiss radiation observations and Open-Meteo forecast data, calculates GHI forecast-risk features, builds a single report-data CSV, creates interactive HTML reports, and emails the reports from a Raspberry Pi.

## Pipeline Overview

The main pipeline is orchestrated by `main.py`.

Each scheduled run does this:

1. Updates MeteoSwiss station radiation data when yesterday is not available yet.
2. Downloads fresh Open-Meteo ensemble and previous-run forecast data for today through today + 5 days.
3. Updates the full-year Open-Meteo station forecast CSV.
4. Runs `PV_Forecast_uncertainty_calc.py` to rebuild the station-level and Switzerland-aggregated GHI forecast files.
5. Runs `GHI_forecast_risk_model_calc.py` to update the single report-data CSV:
   - `output/GHI_forecast_risk_report_data.csv`
6. Runs `GHI_forecast_risk_plot_html.py` twice:
   - one report with scatterplots
   - one report without scatterplots
   An example report can be found here: https://patrik-t-projects.github.io/ghi-forecast-risk-pi/GHI_forecast_risk_example.html
7. Sends both HTML reports in one email.

The scheduler runs at `06:00`, `09:00`, and `12:00` Europe/Zurich time. A failed slot is retried every 30 seconds until it succeeds.

## Data Flow

The report is built from one central CSV:

```text
output/GHI_forecast_risk_report_data.csv
```

This CSV is generated by `GHI_forecast_risk_model_calc.py` from:

```text
data/station_radiation_CH_interp_year_2026.csv
data/station_metrics_CH_interp_year_2026.csv
```

It contains all values needed by the HTML report, including:

- MeteoSwiss measured GHI
- ICON1 previous-day model runs
- forecast-error metrics
- risk-model feature columns
- predicted hourly risk scores
- fitted hourly risk scores where actual forecast errors exist
- q95 forecast-error bands
- daily predicted Kendall-tau performance
- historical fitted Kendall-tau performance
- optional Swissgrid imbalance data when available

For ICON1 risk features, missing ICON1 ensemble columns are filled from the corresponding ICON2 columns where available:

- `ICON1 mean`
- `ICON1 std`
- `ICON1 min`
- `ICON1 max`
- `ICON1 ens spread`
- `ICON1 CC ens spread`

Future rows can contain forecast values, feature values, predicted risk scores, and q95 bands even when actual MeteoSwiss GHI, forecast errors, imbalance data, and fitted-performance values are not available yet.

## Project Structure

- `main.py` - scheduled entry point and pipeline orchestration.
- `config.py` - private local configuration for paths and email credentials.
- `config_example.py` - public template for required configuration values.
- `email_report.py` - sends one or more HTML reports by email.
- `MeteoSwiss_Stations_Weather_data_API.py` - downloads MeteoSwiss station radiation data.
- `API_OpenMeteo_WeatherStations.py` - downloads Open-Meteo station forecast data.
- `PV_Forecast_uncertainty_calc.py` - creates the station radiation and forecast-error CSV files.
- `GHI_forecast_risk_model_calc.py` - updates `output/GHI_forecast_risk_report_data.csv`.
- `GHI_forecast_risk_plot_html.py` - creates the interactive GHI forecast-risk HTML reports from the report-data CSV.
- `GHI_forecast_risk_Fit_Performance.py` - standalone fitting/performance script for diagnostics.
- `manual_update_hourly_performance_tracking.py` - helper/backfill script, not part of the daily pipeline.
- `GHI_forecast_risk_functions.py`, `PV_forecast_uncertainty_functions.py`, `OpenMeteo_API_functions.py` - helper functions.

## Main Outputs

The report-data CSV:

```text
output/GHI_forecast_risk_report_data.csv
```

The HTML reports:

```text
output/GHI_forecast_risk_ICON1_<START_DATE>_to_<END_DATE>_with_scatterplots.html
output/GHI_forecast_risk_ICON1_<START_DATE>_to_<END_DATE>_without_scatterplots.html
```

Both HTML reports are attached to the daily email.

## Setup

Install Python dependencies:

```bash
pip install -r requirements.txt
```

Create your private config:

```bash
cp config_example.py config.py
```

Then edit `config.py` for your Raspberry Pi paths, email sender, email recipient, and SMTP app password.

## Configuration

Keep `config.py` private. It contains machine-specific paths and can contain email credentials.

Required values:

- `DATA_DIR` - folder containing input and generated CSV data.
- `REPORTS_DIR` - folder where HTML reports and report-data CSVs are written.
- `DOWNLOADS_DIR` - folder containing Swissgrid download files.
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD` - SMTP settings.
- `EMAIL_FROM`, `EMAIL_TO` - report sender and recipient.

Use `config_example.py` as the public, non-sensitive template.

## Run

Run the scheduler:

```bash
python main.py
```

Run the report-data update manually:

```bash
python GHI_forecast_risk_model_calc.py
```

Run individual scripts directly only when updating or debugging one pipeline step.

## GitHub Upload Notes

Do not upload your real `config.py`, local data files, generated reports, logs, or credentials. Upload `config_example.py` instead so others can create their own local config.

Good files to upload:

- Python source files, except private `config.py`
- `config_example.py`
- `README.md`
- `requirements.txt`

Keep private:

- `config.py`
- data folders
- output/report folders
- email passwords and app passwords
